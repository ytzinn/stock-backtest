## 1. 이번 세션 진행 순서

1. **STEP 3(신호분리)**: `D_pbr_only`(순수 1/PBR 랭킹, RIM 없음) 추가 — RIM이 R6·1/PBR과
   독립적으로 랜덤 대비 +5%p대 알파를 낸다는 것을 확인
2. **STEP 3B(FactorScreener 원인규명)**: 단일팩터 프리필터+RIM 진단 4개(`E_rev_only` 등) →
   성장성·수익성 팩터가 RIM 알파를 구조적으로 훼손함을 확인, **FactorScreener 전체 폐기 결정**
3. **STEP 3B 후속**: `D_factor_only`(4팩터 컴포지트 단독, RIM 없음)가 랜덤보다도 낮음을 확인 +
   교차상관 분석으로 "위치(프리필터) 문제가 아니라 팩터 구성 자체의 문제"임을 확정
4. **상장폐지 haircut 버그**: 저장된 전체 ablation 결과에서 haircut이 단 한 번도 작동하지 않았음을
   발견 → 수정 → 전체 19개 시나리오 재실행 → RIM 유효성 판정 재역전
5. **StabilityFilter 검증**: R1~R5 leave-one-out + R2/R3/R4 조합 검증(채택 파이프라인 기준) →
   **R2·R3 폐기**, 채택 파이프라인 성과 개선

## 2. FactorScreener 폐기

### 2-1. 단일팩터 진단 (D_rim_only 11.66% 대비, 상폐버그 수정 후 최종 수치)

| 시나리오 (단일팩터 프리필터 + RIM) | CAGR (21구간) | vs D_rim_only |
|---|---:|---:|
| **E_pbr_only** (1/PBR만) | **13.19%** | **+1.53%p (유일하게 개선)** |
| E_op_only (영업이익YoY만) | 7.41% | -4.25%p |
| E_rev_only (매출YoY만) | 2.69% | -8.97%p |
| **E_gpa_only** (GP/A만) | **-1.26%** | **-12.92%p (최악)** |

### 2-2. 컴포지트 단독 효과 + 원인 규명

- `D_factor_only`(4팩터 합산 점수, RIM 없이 직접 선정): **4.72%** — 랜덤(`C_no_r6` 5.23%)보다도 낮음
- 3개 시점(2020/2022/2024년 4월) 교차상관 분석: 매출YoY·영업이익YoY는 `inv_pbr`과 거의 무상관(≈0),
  GP/A만 약한 음의 상관(-0.04~-0.21). 즉 "고성장주=고PBR"이 아니라 **작년 YoY 성장률 자체가
  미래 수익률 예측력이 약하고, R6 없이는 일회성·레버리지성 성장까지 포함**되는 것이 근본 원인.

### 2-3. 결정

FactorScreener **전체 폐기**. Universe 필터는 Hard → Stability → Momentum → RIM 3단계로 확정.
코드(`factor_screener.py`, `ablation.py`의 `E_*`/`G_*`)는 삭제하지 않고 실험 기록으로 보존.

## 3. 상장폐지 Haircut 버그

### 3-1. 발견 경위

저장된 15개 ablation 시나리오 JSON 전부에서 `cagr == cagr_optimistic == cagr_conservative`
(상폐 낙관/기준/보수 3-시나리오가 단 한 번도 서로 달랐던 적 없음)를 발견. `D_rim_only`가 실제
보유했던 상폐 7종목(090730 2018-07-17, 134780 2020-07-02, 066110 2022-04-06, 102280 2025-11-28,
001880 2024-03-04, 052670 2026-02-23, 042670 2026-01-26)으로 재현.

### 3-2. 근본 원인

`engine._calc_period_return()`이 `get_close_price(end_date) is None`을 상폐 트리거로 썼으나,
`get_close_price()`는 `date<=as_of` 최신값을 반환해 상폐로 가격이 끊겨도 절대 `None`이 되지
않았음 — haircut 분기가 도달 불가능한 코드였음. `is_delisted_at()`(`stock_listing_events` 기준,
가격 유무와 무관) 명시 체크로 교체(`backtest/engine.py`, `scripts/export_portfolios.py` 동일 수정).

### 3-3. 수정 전후 영향

| 시나리오 | 수정 전 | 수정 후 | 비고 |
|---|---:|---:|---|
| D_rim_only | 11.99% | **11.66%** | 상폐 7종목 보유분 반영 |
| D_no_r6 | 10.77% | **10.28%** | |
| D_pbr_only | 8.92% | **8.61%** | |
| E_op_only | 7.56% | **7.41%** | |
| E_gpa_only | -0.82% | **-1.26%** | 더 나빠짐 |
| E_pbr_only | 13.53% | **13.19%** | |
| G_full | 9.23% | **9.10%** | |
| C_stability_random(중앙) | 6.80% | **6.70%** | p95: 11.94%→11.81% |
| C_no_r6(중앙) | 5.41% | **5.23%** | |
| F_momentum_rim / F_no_r6 / H_no_stability / G_no_r6 / E_screener_rim / E_rev_only / D_factor_only / E_no_r6 | — | **변화 없음** | 상폐 종목 미보유 |

**판정 변화**: D≥C_p95(RIM 유효성)가 ✅(11.99%≥11.94%, +0.05%p)에서 **❌(11.66%<11.81%, -0.15%p)**로
재역전. 단, F_momentum_rim은 C_p95를 여전히 크게 상회하고 STEP 3 신호분리 결론(RIM이 R6·1/PBR과
독립적으로 랜덤 대비 +5%p대 알파)도 불변이라 RIM 자체를 폐기할 근거는 아님.

랜덤 500회 median이 D_rim_only보다 덜 움직인 이유: 랜덤은 500회 평균이라 상폐 종목 하나를 물어도
스무딩되지만, D_rim_only는 21구간짜리 단일 경로라 실제 보유했던 상폐 종목의 영향이 희석 없이
그대로 실림 — RIM이 "저평가 가치 함정(부실주)"을 랜덤보다 더 많이 물었을 가능성을 시사.

## 4. StabilityFilter R1~R6 개별 검증

### 4-1. H_no_stability 교란 해소

기존 `H_no_stability`는 `use_stability=False` **+ `use_screener=True`**라 stability·screener
두 축이 동시에 달라 F 대비 순수 stability 기여를 볼 수 없었음. `F_no_stability_clean`/
`D_no_stability`(screener 없이 stability만 제거)로 재검증:

| 비교 | CAGR | MDD | Sharpe |
|---|---:|---:|---:|
| D_rim_only (Stability 있음) | 11.66% | -33.9% | 0.423 |
| D_no_stability (Stability 없음, RIM 경로) | 10.68% | **-41.4%** | 0.378 |
| **F_momentum_rim (Stability 있음, 채택)** | **14.63%** | **-32.6%** | **0.508** |
| **F_no_stability_clean (Stability 없음, 채택)** | **13.04%** | **-31.3%** | **0.513** |

모멘텀이 같이 있는 F 경로에서는 Stability를 빼도 MDD·Sharpe가 오히려 소폭 개선됨 —
`H_no_stability`의 MDD 급등(-37.7%)은 Stability 부재가 아니라 **screener 잔존 때문**이었음이 확정.
CAGR 기준으로는 Stability가 여전히 +1.0~1.6%p 기여.

### 4-2. R1~R5 leave-one-out (D_rim_only 11.66% 대비)

| 룰 | 제외 시 CAGR | 제외 시 MDD | 판정 |
|---|---:|---:|---|
| R1 (부채비율) | 11.01% (-0.65%p) | -36.7%(악화) | ✅ 유효 — 수익·리스크 둘 다 개선 |
| **R2 (차입금비율)** | **11.66% (변화 없음)** | -33.9%(동일) | ❌ **완전히 무력** — 이 백테스트 기간 내내 단 한 번도 걸린 적 없음 |
| **R3 (매출역성장)** | **12.19% (+0.53%p, 더 좋아짐)** | **-30.1%(개선)** | ❌ **역효과** — 빼는 게 오히려 나음 |
| R4 (영업CF 2년연속음수) | 11.66% (거의 동일) | -33.9%(동일) | ⚠️ 거의 무력이나 유지 |
| R5 (차입 운영) | 11.46% (-0.20%p) | -34.6%(소폭악화) | ✅ 약하게 유효 |

### 4-3. R2/R3/R4 조합 검증 (채택 파이프라인 F 기준)

R1·R2가 둘 다 부채/차입금 관련이라 R2가 R1에 종속(중복)되는지 확인:

| 시나리오 | CAGR | MDD | Sharpe | 비고 |
|---|---:|---:|---:|---|
| F_momentum_rim (기준) | 14.63% | -32.64% | 0.508 | |
| F_no_r2 | **14.63%** | -32.64% | 0.508 | 기준과 완전히 동일 |
| F_no_r3 | 15.27% | -28.68% | 0.531 | |
| F_no_r4 | 14.71% | -32.64% | 0.511 | |
| F_no_r2r3 | **15.27%** | -28.68% | 0.531 | **F_no_r3와 완전히 동일** |
| F_no_r2r4 | **14.71%** | -32.64% | 0.511 | **F_no_r4와 완전히 동일** |
| F_no_r3r4 | 15.35% | -28.68% | 0.534 | |
| F_no_r2r3r4 | **15.35%** | -28.68% | 0.534 | **F_no_r3r4와 완전히 동일** |

R2는 R3·R4와 어떤 조합에서도 결과에 전혀 영향을 주지 않음 — **R1에 완전히 종속되는 사족**임이 확정.
R3·R4 제거 효과는 서로 가산적(상호작용 없음).

### 4-4. 결정

**R2·R3 폐기**, R1/R4/R5/R6만 유지. R4는 개별 기여가 미미하나(±0.01%p) 경제적 논리(영업CF
2년 연속 음수)가 유효해 보수적으로 유지. 코드(`stability_filter.py`의 R2/R3 판정)는 삭제하지
않고 `active_rules`로 비활성화만 — 향후 재검토 가능하도록 보존.
