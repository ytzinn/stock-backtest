## 1. 배경 — 16.28%는 아직 자기 관문을 치르지 않았다

**[검증된 사실]**

1. RIM에는 D ≥ C_p95 관문을 요구했고 RIM은 −2.80%p로 탈락했다(07-18, CORR-GATE-003 반영 후 C_p95=12.34%). 같은 엄격함이 채택 후보에 적용된 적이 없다.
2. 현행 `C_stability_random`은 HARD + Stability(전체 룰)만 통과시키며 **모멘텀을 통과시키지 않고**, 룰 구성도 채택안의 {R1,R2,R5,R6}과 다르다. 따라서 F_pbr_no_r3r4 16.28%에서 "필터 유니버스 축소 효과"와 "1/PBR 랭킹 고유 기여"가 분리되지 않는다.
3. 07-16 강건성 분석(12/22 승률, LOO 반전, top-k, Jaccard 0.33)은 PIT 이전 holdings 기반이다. 07-18 산출물 목록에 holdings 재생성 기록이 없다. **기존 강건성 수치는 방향성 참고만 가능, 최종 근거 인용 금지.**

**[Claude 의견]** RIM 폐기가 확정 방향인 이상, 재발행 강건성의 1차 비교축은 "PBR vs RIM"이 아니라 **"F_pbr_no_r3r4 vs 정합 대조군 vs 적격 유니버스 동일가중"**이다. PBR vs RIM 쌍대 재실행은 폐기 결정의 기록용 부속으로 격하한다.

## 3. 정합 대조군 신설 (`backtest/ablation.py`)

### 3-1. `C_pbr_path_random`

- 구성: HARD + Stability{R1,R2,R5,R6} + MomentumFilter **통과 풀**에서 무작위 20종목, 동일가중
- 반복: **1,000회** (기존 500회 대비 증가 — p95 추정 안정화). 풀은 리밸런싱일당 1회 구축, 추출만 반복
- `random_seed`를 summary에 기록 (재현성)
- 산출: CAGR 분포의 median / p5 / p95 + F_pbr_no_r3r4의 percentile
- **실행 경로 (2026-07-19 구현 검토, 사용자 확정)**: 풀 캐시 fast-path — 리밸런싱일당
  풀 1회 구축 후, 추첨별 수익률·turnover는 `engine._calc_period_return`/`_calc_turnover`를
  **그대로 재사용**해 계산한다 (전체 엔진 1,000회 재실행 아님). **등가성 게이트 필수**:
  동일 seed 추첨 1회를 전체 엔진 실행과 대조해 CAGR·구간 수익률 일치 확인 후 본 실행.
- 추첨별 **구간 수익률·종목별 기여**도 holdings와 함께 저장 — G3′/G4′ 귀무분포
  계산에서 재계산 대신 직접 사용.

### 3-2. `U_pbr_path_ew` — 적격 유니버스 동일가중

- 구성: 위와 동일한 필터 통과 **전 종목** 동일가중 (랭킹 없음)
- 구현 노트 (2026-07-19): `build_portfolio`의 n_stocks 상한 때문에 "전 종목" 모드가
  현재 없음 — 상한 해제 옵션의 소규모 확장 필요 (기존 태그 무영향 확인 필수)
- 이것이 미결이던 **1차 KPI 벤치마크(기존 Option 2)의 확정안**이다. **[Claude 의견]** "같은 필터를 통과한 유니버스를 그냥 다 사는 것 대비 1/PBR 상위 20 선별이 무엇을 더하는가"가 전략의 존재 이유를 묻는 가장 공정한 질문이며, KOSPI(스타일 불일치)·KOSDAQ(너무 약한 상대)의 문제를 동시에 해소한다. 채택 시 MASTER §3 KPI 절 개정은 SPEC_11 이후 일괄.

## 4. 강건성 산출물 일괄 재발행 (07-18 PIT 기준)

비교쌍: **1차 = F_pbr_no_r3r4 vs U_pbr_path_ew**, 2차(기록용) = F_pbr_no_r3r4 vs F_no_r3r4(RIM 쌍대).

1. 기간별 수익률 CSV (완결 20구간, gross/net/벤치마크 3종)
2. 기간별 holdings JSON (§2에서 완료)
3. 종목별 기여도 (구간 × 종목, 기여 = ret/n)

5. Top-1·2·3 기여종목 제거 (양쪽 동일 처리)
6. Jaccard 중복도 + 스피어만 순위상관 (1/PBR 랭킹 vs U_ew 내 실현수익 순위)

## 5. 판정 기준 — 사전 등록 (2026-07-19 사용자 확정, 실행 후 수정 금지)

### 5-1. 하드 게이트 (채택 확정 조건 — 전부 충족)

| # | 기준 | 근거 |
|---|---|---|
| G1 | F_pbr_no_r3r4 CAGR > C_pbr_path_random **p95** | RIM에 요구했던 것과 동일한 분포 관문 |
| G2 | F_pbr_no_r3r4 net CAGR > U_pbr_path_ew net CAGR | 랭킹이 유니버스 축소 이상의 기여 |
| G5 | 일별 MDD가 **−45%**보다 얕음 (SPEC_09 엔진 기준) | 실전 감내 한계선 — 2026-07-19 확정 |

### 5-2. 집중도 진단 (하드 게이트 아님 — 랜덤 귀무분포 대비 상대 평가)

**설계 원리**: 소형주 저PBR 동일가중 전략의 수익은 양의 왜도(소수 대박 종목·구간)가 엔진이다(Bessembinder 2018). 절대 절단 기준("top-2 제거 후에도 승리" 등)은 왜도 전략에게 자기 엔진을 떼라는 요구가 될 수 있고, n=20 관측치에서 임계값의 이론적 근거도 없다. 따라서 집중도는 **같은 풀에서 뽑은 랜덤 포트폴리오의 동일 지표 분포와 비교**해 "랜덤도 다 갖는 왜도 수준을 넘어선 비정상 집중"만 탐지한다.

| # | 진단 | 계산 |
|---|---|---|
| G3′ | 구간 의존도 | F의 LOO 반전 구간 수(vs U_pbr_path_ew 우위 기준)를 계산하고, **랜덤 1,000회 추첨 각각에 동일 계산**을 적용해 귀무분포 구축 → F의 백분위 보고 |
| G4′ | 종목 의존도 | F의 top-1/2/3 기여종목 제거 후 잔여 마진(vs U_pbr_path_ew)을 랜덤 추첨의 동일 지표 분포와 비교 → 백분위 보고 |
| G6′ | 부호 일관성 | 구간별 paired(F − U_ew) 부호검정 p-값 — **n=20으로 검정력 낮음을 보고서에 명기**, 참고용 |
| G7′ | 시계열 안정성 | 표본 전반 10구간 / 후반 10구간 분할 시 G2 방향 유지 여부 — 참고용 |

**사전 등록 경고선**: G3′·G4′에서 F가 랜덤 귀무분포 대비 **하위 10%** 백분위(= 랜덤보다 유별나게 구간·종목 의존적)에 들면 경고 발화.

### 5-3. 판정 규칙

- **G1·G2·G5 전부 통과 + 경고 미발화** → 채택 확정
- **G1·G2·G5 전부 통과 + G3′/G4′ 경고 발화** → 자동 탈락 아님. **"조건부 채택"으로 사용자 상신**: 라이브 자금 규모 축소 + SPEC_11 포워드 관측 우선 축적을 기본 선택지로 제시. 최종 결정은 사용자
- **G1/G2/G5 중 하나라도 미달** → 아래 FAIL 브랜치

**FAIL 브랜치 사전 등록**: 채택 보류. **그리드 재개·룰 재조합 금지** (07-16 §7 in-sample 누적 경고 유지). 실패 게이트별 진단만 수행 — G1/G2 실패 → 랭킹 무가치 판정, U_pbr_path_ew 자체를 필터-온리 전략 후보로 검토. G5 실패 → 채택 보류 상태로 SPEC_11 라이브 포워드 관측만 축적. 어떤 경우든 결과는 보고서로 발행하고 다음 행동은 사용자 결정.

**참고 기록 (판정 비사용)**: percentile 정확값, KOSPI/KOSDAQ 승률, RIM 쌍대 재실행 결과, G6′/G7′ 수치.
